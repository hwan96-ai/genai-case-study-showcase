A07 — Sapie Reco 배치 추천 엔진 (idus/backpackr)

public-candidate 델리버리 리드 · 총괄 참여

idus(backpackr) 핸드메이드 이커머스 대상 LLM 기반 배치 상품 추천 엔진 — Airflow가 오케스트레이션하고 AWS Batch가 분산 실행하는 구조 위에서, Bedrock Claude 모델을 역할별로 계층화해 약 18,000개 상품의 테마 생성·추천 랭킹을 야간 배치로 수행. 실제 상용 서비스로 공개 출시(보도자료 존재).

역할을 정직하게: 프로젝트 전체 진행과 아키텍처 방향을 총괄하는 델리버리 리드로 참여했으며, 구현 커밋의 다수는 동료 엔지니어의 작업입니다. 이 페이지는 단독 구현 주장이 아니라, 총괄 참여자로서 시스템 구조와 엔지니어링 결정을 설명할 수 있다는 증거로 작성되었습니다.

두 개의 DAG — Airflow 2.10.5(CeleryExecutor)는 스케줄·의존성·재시도만, 무거운 연산은 AWS Batch로 위임

① reco-theme — 키워드 → 테마 생성 파이프라인

키워드상품 ~18,000개 기준 키워드 450개 / 카테고리 30개로 동적 스케일링
→
테마 생성Claude 3.5 Sonnet · 68줄 시스템 프롬프트(좋은/나쁜 예시 + 시즌성·용도 분류 기준 명문화)
→
테마 산출임베딩: Amazon Titan · 벡터스토어: ChromaDB(cosine, HNSW M=64)

② reco-pick — occasion 분류 → 테마 분류 → 추천 랭킹 · 장기 배치 특성을 반영한 48시간 타임아웃 운용

occasion 분류Claude 3.5 Haiku · 대량 호출 단계
→
테마 분류Claude 3.5 Haiku
→
추천 랭킹Claude 3.5 Haiku · 계층적 재랭킹 · 실패 시 CTA 정렬 폴백
→
추천 결과AB 테스트 그룹 라우팅 지원 — 추천 전략 변경을 실험 가능한 구조로

인프라는 AWS CDK/CloudFormation 기반 IaC — Batch 컴퓨트 환경·잡 정의를 코드로 관리해 워커 자원과 LLM 배치 연산의 수명주기를 분리했습니다.

모델 비용 계층화 — 호출량이 많은 단계에 저비용 모델 배치

  • 상품 ~18,000개
  • 배치 사이즈 1000
  • 병렬 워커 10
  • 재시도 3회 + 지수 백오프
  • 48시간 타임아웃
  • 키워드 450 / 카테고리 30

실패 복원력 — 48시간 배치에서 단일 호출 실패가 전체를 죽이지 않도록

LLM 랭킹 호출Claude 3.5 Haiku
→
실패 시 재시도3회 + 지수 백오프
→
최종 실패장시간 배치는 부분 실패가 기본 전제
→
CTA(클릭률) 정렬 폴백비LLM이지만 합리적인 순위 보장 — graceful degradation

계층적 재랭킹 — LLM 컨텍스트 길이·비용 한계를 토너먼트 구조로 우회

50개 초과 배치한 번에 랭킹하기엔 컨텍스트·비용 부담
→
서브배치 분할
→
서브배치별 상위 20% 선별
→
최종 재랭킹컨텍스트·비용을 묶으면서 랭킹 품질 유지
공개 보도자료 (뉴스와이어) ↗ 실제 상용 서비스 공개 출시의 공개 근거 — 이 페이지의 유일한 외부 링크

스택 · Airflow 2.10.5 (CeleryExecutor) · AWS Batch · AWS CDK/CloudFormation IaC · AWS Bedrock — Claude 3.5 Sonnet(테마 생성) / Claude 3.5 Haiku(분류·랭킹) · Amazon Titan 임베딩 · ChromaDB (cosine, HNSW M=64) · TokenTracker 비용 집계

정직한 한계 — 숨기지 않습니다

  • A07 · 정량 성과 미보유 — 전환율·CTR 개선 등 성과 지표는 비공개이거나 미보유. 이 페이지는 성과 수치를 주장하지 않습니다.
  • A07 · 역할은 총괄 참여 — 구현 기여의 다수는 동료 엔지니어. 본인 기여는 델리버리 총괄·오케스트레이션 관점으로 한정해 서술합니다.
  • A07 · 공개 코드 없음 — 소스는 비공개 사본 레포로 외부 검증 가능한 공개 코드가 없으며, 출시 사실의 공개 근거는 보도자료입니다.

Sapie Reco·idus(backpackr) 실명은 승인된 공개 범위이며, 정량 성과 수치는 보유하지 않아 주장하지 않습니다 — 역할(총괄 참여)과 공개 근거(보도자료)의 경계를 그대로 기재했습니다.
