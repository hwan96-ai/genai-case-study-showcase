A03 — Sapie-Braille 에이전트 확장

interview-only

솔트웨어 점자 접근성 솔루션 Sapie-Braille — 3인 팀 팀장 리드 + 최종 행사 대응 기관별 에이전트 3종 단독 개발

기존 Sapie-Braille 서비스(선행 자산) 위에 설정 기반 멀티 테넌트 에이전트 서버를 확장한 작업입니다. “코드 수정 없이 설정으로 기관을 온보딩”이 확장 설계의 핵심이며, 제품 전체에 대한 소유권은 주장하지 않습니다 — 팀 리드와 에이전트 확장이 직접 수행한 기여 범위입니다.

진행 타임라인

  1. 1 기존 Sapie-Braille 서비스 점자 변환 코어 등 선행 자산 — 솔트웨어의 수상 이력이 있는 상용 솔루션
  2. 2 3인 팀 팀장 리드 일정·산출물 관리, 설계 방향, 고객 커뮤니케이션용 문서 등 프로젝트 문서 전반
  3. 3 최종 행사(시연) 대응 행사 일정에 맞춘 기관별 에이전트 추가 개발 요구 발생
  4. 4 에이전트 3종 단독 개발 기관별 YAML 설정 + 코퍼스 구성 + 독립 벡터 인덱스까지 혼자 추가
  5. 5 시연 탑재 + 절차 표준화 기관 3곳용 에이전트 완성·탑재, 신규 기관 추가가 “YAML 설정 + 코퍼스 등록”으로 표준화

정확히 말하면 협업형 멀티 에이전트가 아니라 설정 기반 멀티 테넌트 에이전트 서버입니다 — 면접에서 이 구분을 먼저 말하는 것이 정직한 포지션. 배포는 Dockerfile + EC2 셋업 스크립트 + SSM 환경변수 로더로 실제 AWS에 올려 운영했고, Bedrock 클라이언트 등 단위 테스트가 존재합니다.

역할과 기여 경계 — 무엇이 내 것이고 무엇이 선행 자산인지

역할 1

팀장 (3인 팀)

일정·산출물 관리, 설계 방향 결정, 고객 커뮤니케이션용 문서를 포함한 프로젝트 문서 전반 리드.

역할 2

에이전트 3종 단독 개발

최종 행사 대응으로 기관별 에이전트 3개를 혼자 추가 — 기관별 설정, 코퍼스 구성, 독립 벡터 인덱스까지.

경계

선행 자산은 선행 자산으로

점자 변환 코어 등 기존 Sapie-Braille 서비스는 선행 자산 — full product ownership을 주장하지 않습니다.

  • AWS Bedrock (Claude)
  • LangChain
  • Chroma (S3 동기화)
  • OpenAI 임베딩
  • FastAPI
  • EC2 + SSM 배포 실증
기여 경계 명시: 기존 서비스 위 확장으로, 제품 전체 소유권은 주장하지 않습니다. Sapie-Braille 제품명·솔트웨어 소속은 공개 승인 항목이며, 도입/시연 대상 기관 실명·기관 제공 원본 데이터·구현 코드 발췌는 공개하지 않습니다.

스택 · AWS Bedrock(Claude) · LangChain · Chroma 벡터스토어(S3 동기화) · OpenAI 임베딩 · FastAPI · Dockerfile + EC2 셋업 스크립트 + SSM 환경변수 로더

정직한 한계 — 숨기지 않습니다

  • A03 · 기여 범위 — 기존 상용 서비스 위의 에이전트 확장 기여만 다룸. 점자 변환 코어 등은 선행 자산이며 full product ownership을 주장하지 않음.
  • A03 · 정량 지표 부재 — 결과는 행사 시연 탑재와 온보딩 절차 표준화 중심으로, 공개 가능한 정량 벤치마크 수치 없음.
  • A03 · 공개 경계 — 도입/시연 기관 실명, 기관 제공 원본 데이터, 구현 코드 발췌는 비공개 (interview-only).

고객사·콜봇 플랫폼·월패드 제조사명은 익명 처리했으며, 모든 성능 수치는 자체 벤치마크 산출물 기준입니다 — 최종 선정 탈락 사실을 포함해 결과를 있는 그대로 기재했습니다.
