A04 — CardFollow: 명함 기반 세일즈킷 이메일 에이전트

interview-only · 운영 배포 증거 (로그인 필요)

명함 사진 → OCR(mock↔gpt-4o-mini Vision) → 사람 검수 → Microsoft Graph 발송 · 3주 단독 풀스택(FastAPI+React/TS, 라우터 10 · 페이지 9)

핵심 차별점은 기능이 아니라 발송 경계 설계 — “자동화는 의사결정 지원이지 자율 아웃리치가 아니다”를 코드 수준에서 증명했습니다.

엔드 투 엔드 플로 — 사람 검수와 게이트가 실발송 앞을 항상 막아섭니다

① 명함 사진업로드 — 라이브 데모는 mock OCR 합성 데이터
→
② OCRmock ↔ gpt-4o-mini Vision 전환
→
③ 사람 검수reviewed 강제 — 검수 통과 연락처만 다음 단계로
→
④ 이메일 초안세일즈킷 초안 생성 · 발송 전 확인
→
⑤ SEND_MODE + 7중 게이트아래 사다리·게이트 전부 통과해야 발송 경로 진입
→
⑥ Microsoft Graph 발송게이트 통과 1통 단위만 실발송

SEND_MODE 4단계 사다리 — 단계마다 외부 영향 범위가 한 칸씩만 넓어짐

test
외부 호출 0건
렌더 + 로그만. 라이브 데모가 이 모드로 핀 고정.
manual
Graph 미호출
.eml 다운로드 + 필드 복사 — admin consent 없는 조직에도 가치를 먼저 전달하는 도입 경로.
draft
초안만 생성
Graph로 Outlook 초안만 — 발송 권한(Mail.Send) 분리로 최소권한 유지.
send
게이트 통과 1통만
아래 7중 게이트를 전부 통과한 1통 단위만 실발송.

실발송 7중 게이트

SEND_MODE=send모드 게이트
ENABLE_AUTO_SEND=true이중 env 게이트

이메일 재검증발송 직전 재확인

정직한 한계 — 숨기지 않습니다

  • A04 · 정량 효과 지표 없음 — 발송 수치는 통제 테스트 1통 + 행사 사용 일화 수준, 확인 토큰은 in-memory(수평 확장 불가, 의도된 제약).

고객사·콜봇 플랫폼·월패드 제조사명은 익명 처리했으며, 모든 성능 수치는 자체 벤치마크 산출물 기준입니다 — 최종 선정 탈락 사실을 포함해 결과를 있는 그대로 기재했습니다.
