A08 — 연성대학교 입학상담 챗봇 + SAPiE 멀티테넌트 RAG 백엔드

public-candidate 단독 설계·구현·운영 배포

카카오 i 채널로 연동된 연성대학교 입학상담 챗봇과 사내 SAPiE 멀티테넌트 챗봇 플랫폼의 FastAPI 백엔드 전체를 커밋 이력 기준 단독으로 설계·구현·운영 — 트랙별로 다른 임베딩/벡터스토어 전략, HWP까지 포함한 테이블 인지 문서 파이프라인, SSE 스트리밍·비용 추적 같은 운영 디테일을 갖춘 실제 대학 운영 RAG 서비스입니다.

이중 RAG 트랙 — 같은 시스템 안에서 트랙별로 다른 임베딩·벡터스토어 전략

연성대 트랙 — 단일 도메인 코퍼스

VoyageAI voyage-3-large + FAISS MMR 검색

  • 검색: FAISS MMR (k=3, fetch_k=20) — 단일 도메인 코퍼스에 검색 다양성 적용
  • 입시 연도(2025/2026)에 강하게 묶인 질문 — Jinja2 도메인 프롬프트로 연도별 가이드 분기
  • 2025→2026 입시 전환을 코드 수정이 아닌 프롬프트 계층 갱신으로 흡수
  • 채널: 카카오 i (콜백 비동기 응답)
SAPiE 트랙 — 멀티테넌트

Bedrock Titan + 사용자별 FAISS · S3 영속화

  • 임베딩: Bedrock Titan → 사용자(테넌트)별 FAISS 벡터스토어
  • S3 영속화 + 온디맨드 다운로드 — 테넌트 격리와 인스턴스 무상태성을 동시에 확보
  • "문서만 올리면 자기만의 챗봇" — 업로드 문서 중간 페이지 텍스트로 샘플 질문 자동 생성(온보딩)
  • 채널: 웹 클라이언트 SSE 스트리밍

LLM은 두 트랙 공통: Bedrock Claude Sonnet 4(inference profile) 메인 + Haiku 3.5 폴백/샘플링. 대화 상태는 MongoDB 이력(최근 10턴)을 LangChain 표준 대신 커스텀 chat history 클래스로 직접 제어.

문서 인입 파이프라인 — 입시 요강처럼 표가 핵심인 문서에서 표 구조를 잃지 않는 것이 검색 품질의 전제

  1. 1 업로드 파일 크기 10MB 제한
  2. 2 PDF 변환 HWP / DOCX / XLSX / PPT → PDF — 한국 공공·교육 도메인 특유의 HWP까지 처리
  3. 3 테이블 인지 추출 pdfplumber / pymupdf4llm — 표 유무에 따라 HTML / Markdown 분기 추출
  4. 4 청킹 RecursiveCharacterTextSplitter — chunk 6000 / overlap 250
  5. 5 임베딩 트랙별 임베딩 모델 (VoyageAI / Bedrock Titan)
  6. 6 FAISS 생성 트랙별 벡터스토어 구축
  7. 7 S3 업로드 영속화 — 온디맨드 다운로드로 무상태 인스턴스 지원

운영 디테일 — PoC가 아니라 운영 서비스라서 들어간 것들

스트리밍

SSE 개행 보존 — 커스텀 딜리미터

SSE는 규격상 개행을 이벤트 구분자로 쓰기 때문에 답변 내 개행이 깨지는 문제 → 커스텀 딜리미터로 개행을 보존하는 스트리밍 프로토콜을 직접 정의.

가용성

세마포어 동시성 제한

STREAM_MAX_CONCURRENCY=2 — 소형 인스턴스에서 LLM 스트리밍 폭주로 전체 서비스가 죽는 것을 막는 운영적 안전장치.

대화 상태

최근 10턴 대화 이력

MongoDB 대화 이력(최근 10턴)을 LangChain 표준 대신 커스텀 chat history 클래스로 직접 제어.

비용 가시성

비용 추적 콜백 3종 + 월별 롤업

토큰 단위 비용 추적 콜백 3종(연성/카카오/웹 채널별) + 사용자·월별 비용 롤업 — 멀티테넌트에서 "누가 얼마 쓰는지"를 처음부터 계측.

  • A08 · 정량 성과 미보유 — 상담 건수·응답 정확도 같은 성과 지표는 미보유. 운영 사실과 코드 수준 증거까지만 주장하고 효과 수치는 만들지 않습니다.
  • A08 · 체계적 RAG 평가 부재 — 검색 품질에 대한 골든셋·회귀 평가 체계는 구축되어 있지 않음.
  • A08 · 시크릿 하드코딩 부채 — 비공개 사본 레포에 시크릿이 하드코딩되어 있던 점은 보안 관행상의 부채 — 키 로테이션과 시크릿 매니저 이전이 권고 사항.
